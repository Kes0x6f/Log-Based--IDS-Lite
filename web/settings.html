Detection Sensitivity
Global tuning applied between compiled defaults and per-rule overrides. Per-rule customisations (Rules Manager) always take priority.
live
Threshold multiplier
Scales every rule's compiled attempt-count threshold by this factor. 0.5× fires alerts sooner; 2× requires more attempts before alerting. Has no effect on rules that have a per-rule threshold override.
Global window override
Replaces the sliding window duration for every rule that does not have a per-rule window override. Shorter windows reduce alert lag but increase noise; longer windows suppress transient spikes.
Global cooldown override
Replaces the cooldown (minimum gap between repeated alerts) for every rule without a per-rule cooldown override. Increase to suppress alert storms during heavy attacks; decrease for faster re-alerting.
Resolution order: compiled default → global sensitivity → per-rule override. Changes take effect immediately (cache is invalidated on save). Rules whose DisplayName has not been declared yet (pre-Phase 2.5 rules) are unaffected by the threshold multiplier and window/cooldown overrides.

Retention period
Used by the manual prune button below. Automatic scheduled pruning is not yet wired — use the prune button or set a max-rows cap instead.
Max alert rows
Hard cap on total rows — oldest alerts are pruned automatically after each new alert is inserted when this limit is exceeded.
Manual prune
Check how many alerts are older than the retention period, then delete them immediately.
Retention period and max-rows are persisted in the database. The max-rows cap is enforced automatically on every new alert insert. Manual prune uses DELETE /alerts/prune?days=N.
